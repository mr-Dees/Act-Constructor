# Ручной QA: удаление таблиц рисков

Дата составления: 2026-05-29  
Ветка: `docx-export-redesign` (функциональность реализована в коммитах этой ветки)

---

## Предусловия

1. Приложение запущено (`uvicorn app.main:app`), доступно по `http://localhost:8005` (или актуальному адресу).
2. Открыть любой **процессный** акт (есть 5 разделов; для воспроизведения подходит акт с пустым разделом 5).
3. Убедиться, что в разделе 5 нет таблиц рисков (чистый старт).

---

## Сценарий 1: каждый из 4 типов риск-таблицы можно удалить

**Цель:** убедиться, что все 4 типа (регуляторный, операционный, налоговый, прочий) успешно удаляются.

**Шаги (повторить для каждого типа):**

1. В дереве раздела 5 правой кнопкой мыши нажать на раздел **«5»** → «Добавить подпункт».  
   *Ожидание: в дереве появляется узел «5.1».*

2. Правой кнопкой мыши нажать на узел **«5.1»** → выбрать тип добавляемой таблицы:
   - «Добавить таблицу регуляторного риска» (kind `regularRisk`)
   - «Добавить таблицу операционного риска» (kind `operationalRisk`)
   - «Добавить таблицу налогового риска» (kind `taxRisk`)
   - «Добавить таблицу прочих рисков» (kind `otherRisk`)  
   *Ожидание: в дереве под «5.1» появляется узел таблицы рисков соответствующего типа.*

3. Правой кнопкой мыши нажать на узел таблицы рисков → «Удалить».  
   *Ожидание: открывается диалог «Удаление элемента».*

4. Нажать кнопку **«Удалить»** в диалоге.  
   *Ожидание:*
   - Диалог закрывается.
   - Узел таблицы рисков исчезает из дерева.
   - Появляется уведомление «Элемент удалён».

5. Убедиться, что узел таблицы рисков действительно **отсутствует** в дереве.

---

## Сценарий 2: свод остаётся, пока на уровне есть другой риск

**Цель:** при удалении одной из нескольких риск-таблиц на уровне сводные таблицы должны сохраниться.

**Шаги:**

1. Создать структуру: раздел «5» → «5.1» → «5.1.1» и «5.1.2».

2. Добавить таблицу регуляторного риска на **«5.1.1»** через контекстное меню.

3. Добавить таблицу регуляторного риска на **«5.1.2»** через контекстное меню.  
   *Ожидание: в дереве под «5.1» появилась сводная таблица (per-point), а под «5» — главная сводная.*

4. Удалить риск-таблицу под **«5.1.1»** (ПКМ → «Удалить» → подтвердить).

5. Проверить состояние дерева:
   - Узел риск-таблицы «5.1.1» **исчез**.
   - Сводная таблица под «5.1» **осталась** (потому что «5.1.2» ещё содержит риск).
   - Главная сводная под «5» **осталась**.

---

## Сценарий 3: свод исчезает при удалении последнего риска на уровне

**Цель:** при удалении последней риск-таблицы на уровне каскадно удаляются связанные сводные таблицы.

**Шаги:**

1. Создать структуру: раздел «5» → «5.1» → «5.1.1».

2. Добавить единственную таблицу регуляторного риска на **«5.1.1»**.  
   *Ожидание: появились per-point и главная сводные таблицы.*

3. Удалить риск-таблицу на «5.1.1» (ПКМ → «Удалить» → подтвердить).

4. Проверить состояние дерева:
   - Узел риск-таблицы **исчез**.
   - Сводная таблица под «5.1» **исчезла** (рисков на 5.X уровне не осталось).
   - Главная сводная под «5» **исчезла** (рисков в §5 не осталось).

---

## Сценарий 4: прочий риск создаёт и держит свод

**Цель:** «прочий» тип риска (не входил в свод до исправления) должен полноправно участвовать в механизме сводных таблиц.

**Шаги:**

1. Создать структуру: раздел «5» → «5.1» → «5.1.1».

2. Добавить **таблицу прочих рисков** на «5.1.1» (ПКМ → «Добавить таблицу прочих рисков»).  
   *Ожидание: появились per-point сводная под «5.1» и главная сводная под «5».*

3. Удалить эту единственную «прочую» таблицу рисков (ПКМ → «Удалить» → подтвердить).  
   *Ожидание:*
   - Узел «прочего» риска **исчез**.
   - Per-point сводная под «5.1» **исчезла**.
   - Главная сводная под «5» **исчезла**.

---

## Сценарий 5: перетаскивание риск-таблицы заблокировано

**Цель:** drag-and-drop для узлов риск-таблицы должен быть запрещён.

**Шаги:**

1. Создать «5.1» и «5.2» под разделом 5.

2. Добавить таблицу регуляторного риска на **«5.1»** (через ПКМ).

3. Попробовать перетащить узел риск-таблицы на «5.2» (захватить узел мышью и перенести).  
   *Ожидание: узел не перемещается. Допустимо любое из поведений:*
   - *cursor запрещает drag (иконка «нельзя»)*;
   - *после отпускания узел остаётся на своём месте под «5.1».*

4. Убедиться, что риск-таблица по-прежнему является дочерней для «5.1», а не для «5.2».

---

## Сценарий 6: структура ячеек риск-таблицы залочена

**Цель:** операции, изменяющие колонки и объединение ячеек, недоступны для риск-таблиц (таблица помечена `protected`).

**Шаги:**

1. Создать «5.1» → добавить таблицу регуляторного риска.

2. Перейти на шаг 2 (нажать на вкладку «Содержание» / цифру 2 в шапке).  
   Кликнуть на «5.1» в дереве, чтобы открыть содержимое.

3. Нажать правой кнопкой мыши на любую ячейку риск-таблицы.  
   *Ожидание: появляется контекстное меню ячейки (`#cellContextMenu`).*

4. Проверить, что следующие пункты **серые (disabled) и некликабельны**:
   - «Объединить ячейки»
   - «Вставить колонку слева»
   - «Вставить колонку справа»
   - «Удалить колонку»

5. Убедиться, что операции со строками («Вставить строку выше/ниже», «Удалить строку») **доступны** (не залочены).

---

## Сценарий 7: диалог удаления последнего риска предупреждает о сводной таблице

**Цель:** при удалении последнего риска на уровне диалог должен честно предупреждать пользователя о том, что сводная таблица тоже будет удалена.

**Шаги:**

1. Создать «5.1» → «5.1.1» → добавить единственную таблицу регуляторного риска на «5.1.1».

2. Правой кнопкой мыши нажать на узел риск-таблицы → «Удалить».  
   *Ожидание: открывается диалог «Удаление элемента».*

3. Прочитать текст сообщения диалога.  
   *Ожидание: в тексте есть слова **«сводная таблица»**. Пример фразы:*  
   > «Будет также удалена сводная таблица по пункту 5.1 и общая сводная таблица, так как других таблиц рисков на уровне не останется.»

4. Нажать **«Отмена»** (убедиться, что отмена работает).  
   *Ожидание: диалог закрылся, риск-таблица осталась в дереве.*

5. Повторить шаг 2 и нажать **«Удалить»**.  
   *Ожидание: диалог закрылся, риск-таблица удалена, сводные таблицы удалены.*
